Howdy All?

Сегодня практически завершил работу над подключением терминала, как... да-да, 
как plugin'а! Теперь желающие могут написать свой модуль терминала на любом 
языке программирования :) Интерфейс функции такой:

==============================================================================
type
  T_OpenTerm = function (comportidx:integer;//номер компорта (с 1-го)
                        baudrateidx:integer;//индекс залочки порта (см. ниже)
                        stopbitidx:integer; //индекс стопового бита (см. ниже)
                        databitidx:integer; //индекс дата-бита (см. ниже)
                        parityidx:integer;  //индекс parity (см. ниже)
                        rts: boolean;       //контроль потока: rts
                        xon: boolean;       //контроль потока: xon
                        dialprefix:PChar;   //префикс набора номера
                        icon: HICON         //дескриптор иконки Радиуса
                        ): Boolean; stdcall;

baudrateidx:
--
(br110, br300, br600, br1200, br2400, br4800, br9600, br14400, br19200, 
br38400, br56000, br57600, br115200, br128000, br256000)

Порядковый номер скорости и есть передаваемый Радиусом индекс. Считается с 0.
--

databitidx:
--
(db4,db5,db6,db7,db8)

Порядковый номер и есть передаваемый Радиусом индекс. Считается с 0.
--

stopbitidx:
--
(sb1, sb15, sb2)

Порядковый номер и есть передаваемый Радиусом индекс. Считается с 0.
--

parityidx:
--
(paNone, paOdd, paEven, paMark, paSpace)

Порядковый номер и есть передаваемый Радиусом индекс. Считается с 0.
--
==============================================================================

В настоящий момент в качестве терминала использована терминалка из Реактора.
Причина использования в качестве внешнего модуля вместо интеграции в сорсы в 
том, что не все модули, из которых состоит эта терминалка имеют исходные коды, 
что противоречит принципам открытых исходников эхотага.
