Настройка протоколов передачи

Секция содержит элементы управления настройками протоколов Гидра и BinkP. Эти настройки предназначены для обеспечения совместимости Радиуса с другими мейлерами и для задействования некоторых новых возможностей.

Параметры Описание 
TxWindow
 Размер окна на передачу для протокола Гидра. 
RxWindow
 Размер окна на прием. 
Short and long name
 Передавать на протоколе Гидра и короткое, и длинное имя файла, как делает большинство мейлеров. 
OEM charset
 Использовать для протокола Гидра кодиовку OEM. Эта установка влияет на корректность приема/передачи файлов, содержащих в именах нелатинские.символы. Кодировку OEM традиционно использует большинство мейлеров. Аргус, в отличие от них, использует кодировку ANSI. 
Debug
 Вести хронологию отладочной информации для протокола Гидра.

Параметры Описание 
Request Crypt
 Запрос шифрованного сеанса связи, совместимого с BinkD. Данный режим поддерживается только на протоколе BinkP. См. также Шифрованный сеанс BinkP 
Request File List
 Запрос у удаленной системы списка передаваемых файлов. Если обе стороны поддерживают эту возможность, то по запросу присылается список файлов в той очередности, в которой они будут присланы. Если очередь состоит из единственного файла, то список не передается. Данная опция включается отдельно для протоколов Гидра и BinkP.
Chat enabled Включить чат. После включения чата во время сессии становится доступной кнопка Open Chat, расположенная в нижней части окна линии почтовой системы, при условии, что удаленная система поддерживает режим чата. Чат работает на протоколах Гидра и BinkP. См. также атомы Enable Chat, Disable Chat.  
Chat bell Задать путь к файлу звукового оповещения о запросе чата удаленной системой. Если файл не указан, оповещения не будет. См. также атом Chat Bell.  
Lamps in tray Вместо значка Радиуса показывать в трее иконку с лампочками, дублирующими сигналы модема.
Ignore CD Игнорировать сигнал CARRIER DETECTED модема.

См. также Флаги настройки протокола Гидра.
