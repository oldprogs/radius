Настройки хронологии

В этой секции задаются настройки, касающиеся записи данных хронологии Радиуса.

Параметр Описание
ODBC logging Ведение хронологии посредством ODBC
Log thread times Вести отчет об активности тредов (threads.txt в каталоге хронологии).

Ведение хронологии посредством ODBC

Для включения этой возможности создайте в ODBC Data Source Administrator (Администратор источников данных ODBC) алиас "ArgusLog". В базе создайте таблицу "LogStrings" со следующими обязательными колонками: три текстового типа - "Logger", "RecTag" и "RecMsg" - и одну типа дата/время "RecTime". Вы можете также создать дополнительную колонку, содержащую автоматически увеличивающийся номер ряда, но убедитесь, что значения этой колонки будет устанавливаться автоматически. Для добавления записей Радиус выполняет следующее SQL-выражение:

 INSERT INTO LogStrings (Logger, RecTag, RecTime, RecMsg) VALUES (?, ?, ?, ?)

Для создания таблицы LogStrings в MS Access можно выполнить следующее SQL-выражение:

       create table LogStrings (
         RecId   counter NOT NULL CONSTRAINT RecIdIndex PRIMARY KEY,
         Logger  char(20) NOT NULL,
         RecTag  char(1) NOT NULL,
         RecTime date NOT NULL,
         RecMsg  char(250) NOT NULL);

Затем в Config/Start-up включите "ODBC logging" и перезагрузите Радиус. Ошибки ODBC будут записываться в файл "odbcerr.log" в каталоге хронологии Радиуса.
