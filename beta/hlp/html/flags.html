Внешние флаги

На закладке Flags диалогового окна Config/Externals настраивается обработка внешних файл-флагов. Наличие / отсутствие файл-флага может приводить к выполнению определенного действия: запуска внешней программы или перезагрузки Радиуса. Для этого в поле Execute command указывается командная строка запуска внешнего приложения, а в поле Flag - полный путь к файл-флагу (с опциональным префиксом), который сопоставляется данному приложению. Если путь не указан, предполагается, что файл-флаг находится в каталоге флагов Радиуса.

Формат флагов (столбец flag):

flag - если существует (создан) флаг, удалить его и выполнить команду
?flag - если существует (создан) флаг, выполнить команду без удаления флага
&flag - если отсутствует (удален) флаг, создать его и выполнить команду
*flag - если отсутствует (удален) флаг, выполнить команду без создания флага

В поле Execute command вместо строки запуска внешнего приложения допускается указывать следующие макросы:

%RESTART<MSEC>%
 Перезагрузить Радиус с паузой в MSEC миллисекунд. Минимальное значение MSEC равно 5000. Если параметр <MSEC> не задан, то он принимается равным 5000. Данный макрос может использоваться с целью удаленного изменения настроек, когда после загрузки нового файла конфигурации необходимо перезапустить Радиус. 
%STARTRAS% /
%STOPRAS%
 Запустить / прервать соединение посредством RAS. Настройки соединения определяются в секции RAS DialUp диалогового окна "Настройки". См. также атом RAS Event.

Примеры:

Flag Execute command Комментарий 
restart.flg
 %RESTART5000% Перезагрузить Радиус с паузой в 5с и удалить флаг 
?D:\FIDO\FLAGS\session.fail
 x:\path\my_service.bat Запустить внешнее приложение (пусть оно само и флаг удаляет)
