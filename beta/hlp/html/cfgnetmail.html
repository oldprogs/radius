Динамическая маршрутизация

Динамическая маршрутизация включается установкой флажка Use dynamic routing.

Схемы маршрутизации задаются в виде строк сетки, имеющей следующие поля: "Via", "From" и "Password". Поле "From" - это список разделенных пробелами адресов узлов назначения. Группы адресов разрешается задавать с помощью символа "звездочка" (*), а знак "диез" (#) в маске a:b/c.# обозначает подчиненные узлы (подхабники) хаба a:b/c. Для исключения адресов из группы используется символ "восклицательный знак" (!).

В поле "Via" назначается адрес узла, через который должна быть отправлена почта для узлов "From". Поле "Password" задает пароль для почтовых пакетов.

Нужно иметь в виду, что, согласно правилам, установленным для FTN-сетей, транзитные почтовые пакеты должны проходить только по специально установленным маршрутам. Узлам не вменяется в обязанность (и не рекомендуется) пропускать "чужие" пакеты. Поэтому первым делом следует удостовериться в том, что узлы, указанные в поле "Via", согласны передавать почтовые пакеты, приходящие от вас.

Ниже приведен пример схемы маршрутизации. В данной схеме указано, что почта для подсети 2:461/* будет отправлена либо через узел 2:461/640, либо через 2:461/700, в зависимости от того, с кем из них раньше произойдет сессия. Почта для узлов зоны 6 будет отправляться только через 2:461/640, а почта для узла 1:123/500 и подсети 2:463/* (кроме узла 2:463/100) - через 2:461/700. При этом почта для любого из узлов будет отправлена ему напрямую, если с ним произойдет сессия (см. последнюю строку), а почту для 2:463/100 разрешается отправлять только лично ему. Рекомендуется всегда указывать в схемах маршрутизации последнюю строку приведенного примера.

Via From Password
2:461/640 2:461/* 6:* password
2:461/700 2:461/* 2:463/* !2:463/100 1:123/500 password
* *  

Следует отметить, что пpи неудачной сессии перепакованные пакеты не запаковываются обратно в исходное состояние, т.к. в начале следующей сессии они все pавно запакуются должным образом.
