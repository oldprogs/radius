Общие параметры

Здесь задаются общие настройки Радиуса.

Параметр Описание Пример
Main address Основной адрес 2:5012/38
Main region Основной регион. Этот параметр используется в дополнение к основному адресу для определения принадлежности вашего узла к конкретной сети. Для России значение региона должно быть равно 50. 50
Synch clock with Возможность синхронизации системного времени локальной машины с временем удаленного узла. Если узел указан, то при каждом сеансе связи с ним Радиус будет проверять и при необходимости корректировать локальное время. Поддерживается 3d- и 4d-формат FTN-адреса удаленного узла. 2:5012/708
Synch time shift Сдвиг в секундах между временем локального часового пояса и часового пояса удаленного узла, с которым производится синхронизация. Этот параметр позволяет синхронизировать время с узлом, расположенным в другом часовом поясе. 0
CPS min bytes Минимальное количество переданных/полученных байт для расчета CPS и прогнозного времени сессии. 16385
CPS min secs Минимальное количество времени в секундах от момента начала сессии для расчета CPS и прогнозного времени сессии. 11
Max BSY age Максимальный срок действия флагов занятости удаленного узла (BSY) в секундах. По истечении указанного срока флаги BSY считаются старыми и могут быть удалены при помощи либо макроса %DELBSY%, либо атома Del Old *.BSY. См. также Буфер отправки системы Binkley. 259200
Max BWZ age Максимальный период, после которого недопринятые файлы будут автоматически удаляться из временного каталога. См. также Панель BadWaZOO. 259200
Max UDL age Максимальный период, после которого будут сняты ручные паузы с прозвонок, установленные из меню Poll / Pause. При установке этого параметра в 0 паузы с прозвонок сниматься не будут.. 0
Outb. rescan Период сканирования каталога отправки в миллисекундах. 2000
Msgs. autoclose Время в секундах, по истечении которого окна предупреждающих сообщений будут автоматически закрываться. 15
Use dynamic outbound Включить режим динамический отправки.  
5d outbound Поддержка 5d буфера отправки системы Binkley.  
Create SESSION.OK fileflag
Create SESSION.FAIL fileflag		 Возможность создавать файл-флаги SESSION.OK, SESSION.FAIL после успешного или неудачного завершения сеанса связи, соответственно. См. тж. пункт Отсоединение раздела Окно линии почтовой системы.  
Use Nodelist Data Проверять наличие флагов TCP/IP в списке узлов. При включенной опции узлы, у которых таких флагов нет, считаются недоступными по TCP/IP. Радиус воспринимает следующие флаги TCP/IP из списка узлов: IBN, BND, BNP, IFC, ITN, TEL. При выключенной опции поддерживается прежнее поведение: даже если в списке узлов отсутствуют флаги TCP/IP, Радиус будет по умолчанию подставлять флаги IBN, CM. Следует иметь в виду, что все данные списка узлов перекрываются настройками таблицы переопределений IP-узлов. См. также Прозвонка на узел только в его рабочее время.  
Auto compile nodelists Автоматическая компиляция списка узлов при любом изменении.  
Use SPACE to flush timeout Использовать для сброса интервала ожидания клавишу "пробел" вместо Ctrl+PdDn. См. тж. Панель времени ожидания и Горячие клавиши.  
Fixed retry timeout Алгоритм определения реальных значений Stand-off minutes и Retry Seconds. Если этот флажок установлен, берутся точные значения, указанные в окне конфигурации параметров прозвонок. Если не установлен, то Радиус работает по алгоритму, используемому в Аргусе: в качестве реального значения интервала используется случайное число в диапазоне от n*0.5 до n*1.5 (то есть+-50%). Например, при установленном в Retry Seconds значении 100 повторные прозвонки будут происходить со случайным интервалом от 50 до 150 секунд.  
Play Sounds Включить озвучиание событий.  
Close button action Возможность настройки поведения Радиуса при нажатии на кнопку "Закрыть" ("крестика" в правом верхнем углу окна Радиуса):
  • Shutdown - обычное поведение (выход из программы)
  • Ask for shutdown - выдавать окно подтверждения выхода
  • Minimize to tray - сворачивать Радиус в трей
 Shutdown
Min free space Минимально допустимый объем свободного дискового пространства в байтах в парольном каталоге. 100
