Каталог хронологии

Log - в этом каталоге ведется история Менеджера Прозвонок (polls.log), TCP/IP Deamon'а (ipdaemon.log), периодического запуска внешних приложений (cronapps.log), протокол модемных (line.log) и TCP/IP (ip_?.log) линий почтовой системы, протоколы EMSI-данных (emsi-line.log) и результатов обработки скриптов (exec-line.log) для линий, протоколы тарифов (tariff.log) и ошибок модуля тарификации (tariff.err), протокол работы с сервером удаленного доступа (ras.log), отчет об активности тредов (threads.txt), а также дополнительные файлы хронологии.

Имена некоторых файлов хронологии, заданные по умолчанию, могут быть изменены пользователем в группе File Names этой секции.

Дополнительные файлы хронологии

Кроме готовой текстовой хронологии, предназначенной для восприятия системным администратором, Радиус ведет также дополнительные файлы хронологии:

access_log
 Общая хронология переданных файлов в стандарте NCSA 
agent_log
 Общая хронология типов удаленных почтовых систем в стандарте NCSA 
binary_log
 Общая хронология (в двоичном формате) сеансов связи и переданных данных

Файлы access_log и agent_log доступны для анализа любым NCSA-совместимым анализатором статистики.

Формат файла binary_log совместим с используемым в T-Mail © Andy Elkin и BinkD © Dima Maloff. Для обработки результатов этого файла можно воспользоваться утилитой T-Hist © Michael Markowsky (2:5020/378@fidonet).
